Everything code-side is done and pushed. This is the exact, ordered checklist of the steps that
need **your accounts / your hands**. Do them top-to-bottom (later steps depend on earlier ones).
For each: what to click, the link, and **what value to send me** so I wire it into the app.

Legend: 🔴 **required to qualify** · 🟡 strongly recommended (Catalyst-validity) · 🟢 optional polish.

## 0. 🔴 Regenerate fresh data (10 seconds, do this the day you deploy/demo)
The dataset is time-anchored to "today" so the live forecasts/hotspots stay current.
```bash
npm install
npm run data:generate      # refresh data/seed (today-anchored)
npm run validate           # must print: 10 passed, 0 failed
node catalyst/scripts/seed-to-csv.mjs   # refresh data/csv for Data Store import
```
Then `git add -A && git commit -m "refresh data" && git push` so the deployed build has current data.

## 1. 🔴 Claim Catalyst credits (unblocks everything)
1. Open **https://catalyst.zoho.com/promotions.html?cn=KSPH26** — sign up **through this exact link**
   (the KSPH26 coupon is in the URL; signing up elsewhere won't attach it). **Country must = India.**
2. In the Catalyst console → **Settings** (top-right gear) → **Billing** → **Store page** → click the
   **Basic** plan → tick **"Use my existing wallet credit for this transaction"** → **Proceed** and
   finish. You'll get 1,500 credits + $250 applied (confirmation email).
> **Send me:** nothing yet — just tell me **"credits claimed."**

## 2. 🔴 Install the CLI, log in, create the project
```bash
npm install -g zcatalyst-cli
catalyst login          # opens browser; sign in with the SAME Zoho account
```
Then create a project: console → **Create Project** (or `catalyst init` in the repo). Pick data
center **India (IN)** (required — QuickML/Zia are available in IN).
> **Send me:** **"logged in, project = `<project-name>`, datacenter IN"** and the **project ID**
> (shown in the console URL). → I then deploy.

## 3. 🔴 Deploy to Catalyst AppSail (I drive this once you're logged in)
From the repo root:
```bash
catalyst init          # choose AppSail; bind the project
catalyst appsail init  # stack node18; build: npm install && npm run build --workspace=apps/web ;
                       # start: npm run start --workspace=apps/web   (port is auto — start.mjs handles it)
catalyst deploy        # builds, uploads, returns your PUBLIC URL
```
Open the URL in an **incognito** window → confirm it loads (200). That URL is your **Deployed
Solution Link** for the submission form.
> **Send me:** the **live AppSail URL** → I put it in the README + submission brief and smoke-test it.

## 4. 🟡 Catalyst Data Store (make "built on Catalyst" true at runtime)
1. Console → **Cloud Scale → Data Store → Create Table** for each of the 9 tables. Use the **exact
   column names + types** in **`data/csv/_SCHEMA.md`** (generated by the CSV script). Tip: store IDs
   as plain `BigInt`/`VarChar` (skip Foreign-Key columns) to avoid import ordering pain.
2. Import the CSVs (in this order: districts → stations → persons → firs → fir_accused → fir_victim
   → accounts → transactions → associations):
   ```bash
   catalyst ds:import data/csv/districts.csv --table districts
   # …repeat for all 9. The CLI uploads to Stratus + runs the bulk job + prints a skip report.
   ```
   (Dev caps: 5,000 rows/table, 25,000/project — we're ~3.5k total, fine.)
> **Send me:** **"Data Store loaded"** → I flip `NETRA_ENV=catalyst` and wire the routes to read from it.
